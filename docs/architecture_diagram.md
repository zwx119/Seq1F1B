# DeltaNet × Seq1F1B 架构图

## 系统总览（1.3B/2.7B: TP=1, PP=4, DP=2）

```
                              8× A100-80GB
┌─────────────────────────────────────────────────────────────────────────────────┐
│                                                                                 │
│  DP Rank 0:                                                                     │
│  ┌─── PP Stage 0 ──┐ ┌─── PP Stage 1 ──┐ ┌─── PP Stage 2 ──┐ ┌── PP Stage 3 ─┐│
│  │ GPU 0: L 0-5    │→│ GPU 1: L 6-11   │→│ GPU 2: L 12-17  │→│ GPU 3: L 18-23││
│  └─────────────────┘ └─────────────────┘ └─────────────────┘ └────────────────┘│
│                              P2P Send/Recv                                      │
│  DP Rank 1 (同样的模型副本，不同数据):                                            │
│  ┌─── PP Stage 0 ──┐ ┌─── PP Stage 1 ──┐ ┌─── PP Stage 2 ──┐ ┌── PP Stage 3 ─┐│
│  │ GPU 4: L 0-5    │→│ GPU 5: L 6-11   │→│ GPU 6: L 12-17  │→│ GPU 7: L 18-23││
│  └─────────────────┘ └─────────────────┘ └─────────────────┘ └────────────────┘│
│                                                                                 │
│  8 GPUs = PP(4) × TP(1) × DP(2)                                                │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## 并行维度说明

```
┌────────────────────────────────────────────────────────────────────────┐
│                       完整训练配置                                      │
│                                                                        │
│  PP (Pipeline Parallel) = 2                                            │
│  PP (Pipeline Parallel) = 4                                            │
│  ├── 模型层按深度切分到 4 个 GPU 组                                     │
│  ├── PP=4, 24层: L0-5→S0, L6-11→S1, L12-17→S2, L18-23→S3             │
│  └── GPU 间通过 P2P 发送激活值                                          │
│                                                                        │
│  TP (Tensor Parallel) = 1 or 2                                         │
│  ├── 每层的权重矩阵按列/行切分到多张 GPU                                 │
│  ├── TP=2: Q/K/V 的 head 分两组，各 GPU 算一半                          │
│  └── 用 AllReduce 同步结果                                              │
│                                                                        │
│  PP_SP (Pipe-SP / Seq1F1B Splits) = 4                                  │
│  ├── Seq1F1B 的核心：将每个 microbatch 的序列切成 PP_SP 份               │
│  ├── 每份叫一个 "span"（也称 micro-sequence）                           │
│  ├── 调度器交错执行不同 span 的 F/B，减少流水线气泡                      │
│  └── 对 DeltaNet: span 间传递 recurrent state S                        │
│                                                                        │
│  SP (Sequence Parallel, --sequence-parallel) ← Megatron 内置            │
│  ├── TP 的配套优化，不是独立并行维度                                     │
│  ├── LayerNorm/Dropout 沿序列维度分片到 TP 组内各 GPU                    │
│  ├── TP=1 时自动关闭                                                    │
│  └── 目的：减少 TP 组内的激活值显存                                      │
│                                                                        │
│  DP (Data Parallel) = WORLD_SIZE / (PP × TP)                           │
│  ├── 8 GPU, PP=4, TP=1 → DP=2                                         │
│  ├── 8 GPU, PP=4, TP=2 → DP=1 (7B 模型)                               │
│  └── 各 DP rank 处理不同数据，AllReduce 梯度                            │
└────────────────────────────────────────────────────────────────────────┘
```

## 核心架构图：Seq1F1B × DeltaNet 的序列切分与状态传递

```
         输入序列 (seq_length = 32768 tokens)
    ┌────────────────────────────────────────────────────┐
    │  token_0 .............. token_32767                │
    └────────────────────────────────────────────────────┘
                         │
                    PP_SP = 4 切分
                         │
                         ▼
    ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
    │  Span 0  │ │  Span 1  │ │  Span 2  │ │  Span 3  │
    │ 8192 tok │ │ 8192 tok │ │ 8192 tok │ │ 8192 tok │
    └──────────┘ └──────────┘ └──────────┘ └──────────┘
         │             │             │             │
         ▼             ▼             ▼             ▼
    ┌─────────────────────────────────────────────────────────────────────────┐
    │              Seq1F1B 流水线调度器 (PP=4, PP_SP=4)                        │
    │                                                                         │
    │  Stage 0 (GPU 0):  F₀₀ F₀₁ F₀₂ F₀₃ │ B₀₀ F₁₀ │ B₀₁ F₁₁ │ ...        │
    │  Stage 1 (GPU 1):       F₀₀ F₀₁ F₀₂ │ F₀₃ B₀₀ │ F₁₀ B₀₁ │ ...        │
    │  Stage 2 (GPU 2):            F₀₀ F₀₁ │ F₀₂ F₀₃ │ B₀₀ F₁₀ │ ...        │
    │  Stage 3 (GPU 3):                 F₀₀ │ F₀₁ F₀₂ │ F₀₃ B₀₀ │ ...        │
    │                                                                         │
    │  F_ij = microbatch i 的 span j 的 Forward                                │
    │  B_ij = microbatch i 的 span j 的 Backward                               │
    │                                                                         │
    │  关键：连续 span 交错执行，减少气泡                                       │
    │  (GPU 4~7 是 DP Rank 1，执行相同调度但处理不同数据)                       │
    └─────────────────────────────────────────────────────────────────────────┘
```

## DeltaNet 层内的 Chunk 计算（在每个 Span 内部）

```
    Span j (8192 tokens) 进入 DeltaNet 注意力层
    ┌────────────────────────────────────────────────────┐
    │  token_0 .............. token_8191                 │
    └────────────────────────────────────────────────────┘
                         │
              fla chunk_delta_rule
              内部按 BT=64 分 chunk
                         │
                         ▼
    ┌────┐┌────┐┌────┐┌────┐       ┌────┐
    │ c0 ││ c1 ││ c2 ││ c3 │ ...  │c127│  ← 8192/64 = 128 个小 chunk
    │ 64 ││ 64 ││ 64 ││ 64 │       │ 64 │
    └────┘└────┘└────┘└────┘       └────┘
      │      │      │      │          │
      ▼      ▼      ▼      ▼          ▼
    ┌──────────────────────────────────────────┐
    │  chunk 间用 recurrent state S 传递信息    │
    │                                          │
    │  chunk 0:                                │
    │    S₀ = initial_state (从上一个 span)     │
    │    intra-chunk: 用矩阵乘法并行计算        │
    │    inter-chunk: 更新 S₁                   │
    │                                          │
    │  chunk 1:                                │
    │    S₁ → intra-chunk 计算 → S₂             │
    │    ...                                   │
    │                                          │
    │  chunk 127:                              │
    │    S₁₂₇ → 计算 → S_final                 │
    │    S_final 缓存，供下一个 span 使用        │
    └──────────────────────────────────────────┘
```

## 两层嵌套关系的完整视图

```
┌─────────────────────────────────────────────────────────────────────┐
│                     Seq1F1B 调度层 (外层)                            │
│                                                                     │
│  序列 32K tokens                                                    │
│  ─────────────────────────────────                                  │
│  │← Span 0 →│← Span 1 →│← Span 2 →│← Span 3 →│                   │
│  │  8192 tok │  8192 tok │  8192 tok │  8192 tok │                  │
│                                                                     │
│  每个 Span 作为独立的 "micro-sequence" 在流水线中调度                 │
│  Span 间通过 recurrent state S 保持因果连续性                        │
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │               DeltaNet 算子层 (内层)                         │    │
│  │                                                             │    │
│  │  每个 Span (8192 tokens) 内部:                              │    │
│  │  ┌──┐┌──┐┌──┐ ... ┌──┐                                     │    │
│  │  │64││64││64│     │64│  ← BT=64 的 chunk (共 128 个)       │    │
│  │  └──┘└──┘└──┘     └──┘                                     │    │
│  │                                                             │    │
│  │  chunk 间: recurrent state S ∈ R^{d_k × d_v} 精确传递       │    │
│  │  chunk 内: 用矩阵乘法并行计算 (Triton kernel)                │    │
│  │                                                             │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  State 传递链:                                                      │
│  Span 0                Span 1                Span 2                 │
│  ┌──────────┐          ┌──────────┐          ┌──────────┐           │
│  │ c0→c1→...→c127 │ S →│ c0→c1→...→c127 │ S →│ c0→c1→...     │    │
│  │           │final│    │           │final│    │           │         │
│  └──────────┘  │        └──────────┘  │        └──────────┘         │
│                │   .detach()          │   .detach()                  │
│                └──────────────────────└─────────────────────         │
│                                                                     │
│  注意: .detach() 意味着 forward 信息精确传递，                       │
│        但 backward 梯度在 span 边界截断                              │
│        (这是 Seq1F1B 的设计特性，Softmax 版本也如此)                 │
└─────────────────────────────────────────────────────────────────────┘
```

## DeltaNet vs Softmax 的 Span 间状态对比

```
                    Softmax (FlashAttention)               DeltaNet
                    ─────────────────────────              ──────────────────

Span 间状态:         KV cache                              Recurrent state S
                    K: [s, b, h_d]                         S: [H, d_k, d_v]
                    V: [s, b, h_d]                         (常量大小!)

大小随 seq 变化:     O(s × d) 线性增长 ❌                   O(d² ) 固定 ✅

seq=32K, SP=8:      40.8 GB 显存                           35.4 GB 显存
seq=64K, SP=16:     77.0 GB → OOM ❌                       75.3 GB → ✅

DeltaNet state 大小计算:
  d_k = d_v = head_dim = 128
  H = num_heads_per_partition = 16
  S = H × d_k × d_v × bytes = 16 × 128 × 128 × 2 = 512 KB / layer
  24 层 × 512 KB = 12.3 MB (几乎可忽略!)

Softmax KV cache 大小:
  per span: 2 × s_span × H × d × bytes
  seq=32K, SP=8: 2 × 4096 × 16 × 128 × 2 = 32 MB / layer
  24 层 × 32 MB = 768 MB
  还要保存所有之前 span 的 cache → 累计 GB 级
```
